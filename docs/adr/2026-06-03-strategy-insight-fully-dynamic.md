# ADR-020: 戦略診断アドバイスの完全動的化（サイト固有タスク流用）

## 日付
2026-06-03

## 状況

ADR-018 で戦略診断の前半（SoM点・ティア・引用順位）は実測値の動的埋め込みにより定型文を撲滅した。しかし後半の「次の一手」アドバイス文と推奨3アクションは、いまだ SoM 帯ごとの固定テンプレート（`tierAdviceFromSom` の `nextMove` と `ACTIONS_LOW`/`ACTIONS_HIGH`）に依存しており、「何を解析しても同じ後半文・同じ3アクション」という定型文の最後の残骸だった。

一方、ルーブリックAI監査は既に **サイト固有の優先改善タスク**（`ConsultantOutputData.prioritizedTasks`：例 Schema.org実装・H1追加・robots.txt整備）を生成している。これを後半アドバイスに流用すれば、追加のLLM呼び出しなしに完全動的化できる。

## 決定

`StrategyInsightService.describeForQuery` にサイトタスクを受け取るオーバーロードを追加（2引数版は3引数版へ委譲し既存呼び出し・テスト互換）。

- サイトタスクがあれば後半文を「このサイト固有の最優先改善は『〇〇』です。」に動的化し、推奨アクションをタスクベースに置換。
- 無ければ従来の SoM 帯テンプレートへフォールバック（劣化しない）。
- `sanitizeSiteTasks` で 空白除去・重複排除・**帯固定テンプレ文（ACTIONS_LOW/HIGH）の除外**・上限3件。定型文がアドバイスに混入するのを防ぐ。
- クエリ永続化経路（`JobPersistenceService.upsertAuditHistoryForJobQuery`）で、保存対象のコンサル出力JSON（`rawResponse`）から `prioritizedTasks` の title を抽出して渡す。パース不能時は空＝フォールバック。
- ジョブ全体ロールアップ（`rollupJobFromTemplate`→`describeJobRollup`）でも、各監査行に蓄積されたサイト固有アクションを集約して後半を動的化。AI駆動ロールアップ（`DebateAdviceGeneratorService`）は既に動的なため変更なし。

## 理由

- 追加コストゼロ（既存の監査出力を再利用）。核④「1解析＝1チケット／高利益率」を死守。
- 帯テンプレ除外により「サイト固有でない汎用文」がアドバイスに紛れ込むのを構造的に防止。
- 2引数版を残すことで `GeminiResultProcessor`・`resolveForAudit` 等の既存呼び出しとテストを壊さない。
- 代替案「全経路でタスク必須」は監査未完了時に空アドバイスになるため却下（フォールバック必須）。

## 結果

- サイト固有タスクが存在すれば、解析対象が違えば後半文・3アクションも変わる＝定型文の完全撲滅。
- タスク無し時は従来挙動を厳密維持（テストで担保）。
- `rawResponse`（コンサル出力JSON）への暗黙依存が1箇所増えた。パース失敗を握りつぶさずフォールバックする設計で堅牢性を確保。
