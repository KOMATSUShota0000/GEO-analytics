# ADR: 解析結果UIの用語・指標わかりやすさ改善（改Z'の内部指標化）

- 日付: 2026-06-09
- ステータス: 採用
- 関連: CPO仕様 `.cursor/plans/2026-06-09-analysis-ui-clarity.md`

## 背景

オーナーが実機確認中、「解析結果一覧」に可視性系の指標が4つ（GBVS・改Z'・Stage・GEO可視性ランク）並び、**同じ「AI可視性」を点数・偏差・段階で3重に見せている**ため意味が伝わらない、と指摘した。特に：

- **改Z'（modifiedZScore）** は「ジョブ内のSoM中央値からの相対偏差」で、1クエリ解析では IQR=0 となり**必ず0**。その中央値（`medianModifiedZ`）も「中央値からの乖離の中央値」ゆえ定義上ほぼ常に0になり、ユーザーには故障に見える無意味な数値だった。
- **GBVS** は Generative Brand Visibility Score の略で、SoM（Share of Model）の正規化スコアだが、略語が伝わらない。
- **GEO可視性ランク** は AI回答内での引用順位だが、Stage と語感が紛らわしい。

ホワイトラベルでクライアントに提示される画面のため、用語の平易さが信頼に直結する。

## 決定

1. **GBVS → 「SoMスコア」にリネーム**（表示ラベルのみ。値・計算・内部フィールド名は不変）。既存の SoM 表記と統一する。
2. **GEO可視性ランク → 「AI引用順位」にリネーム**し、「AI回答の中で何番目に引用されたか」をツールチップで補足する。
3. **改Z' を UIから撤去し内部指標化する**。解析結果一覧の「改Z'」列、ジョブ戦略診断ブロックの「中央値ベースの改Z'」表示、印刷レポートの「ジョブ中央値（改Z'）」表示を削除。表示専用の `displayJobRollupMedZ` を削除し、`showJobStrategyBlock` の表示条件を「診断文 or 推奨アクションの有無」に是正（従来は常に非nullの改Z'中央値に依存しており実質常時表示だった）。
   - **ただし `modifiedZScore`/`jobMedianModifiedZ` のデータと `rowShowsDetailedStrategy`（中央値から±1σ以上外れたクエリだけ詳細戦略を出す出し分け）・Stage算出（`stageFromModifiedZ`）は内部ロジックとして維持する。** 改Z'は「ユーザーに見せる数値」としては無意味だが、複数クエリ間の外れ値検出には有効なため、裏方では使い続ける。振る舞いは不変。
4. **「評価 N クエリ」をクリックで使用クエリ一覧を開示する**。`AiRecognitionSection` に `queries?: string[]` を追加し、MUI Link＋Collapse でトグル展開。データは親（`JobAnalysisPage`）が既に保持する各クエリ文字列を再利用し、**バックエンド追加・新規APIなし**で実現する。

## 検討した代替案

- **改Z'中央値を「SoM中央値」など別の数値に差し替える**: オーナーが「中央値の数値表示自体が不要」と判断したため、差し替えではなく撤去を採用。
- **改Z'を内部ロジックごと完全撤去し Stage を SoM絶対値ベースで再設計する**: 作業量が大きくリグレッションリスクが高い。今回はUI撤去のみとし、内部指標として温存した。
- **使用クエリ一覧をバックエンドの専用フィールドで返す**: 既に画面内に各クエリ文字列があるため不要。LLM消費・APIを増やさないフロント再利用を選択。

## 既知のトレードオフ

- `showJobStrategyBlock` から改Z'中央値依存を外したことで、「診断文も推奨アクションも無いが改Z'中央値だけある」という稀なケースでは戦略ブロックが非表示になる。診断文・アクションが空のブロックは元々情報価値がないため、これは望ましい挙動として受容する。
- `queries` prop はオプショナルとし、未指定時は従来どおり静的テキスト表示にフォールバックする（旧データ・過渡期の後方互換）。

## 影響

- 変更: `frontend/src/pages/JobAnalysisPage.tsx`・`frontend/src/pages/ReportPrintPage.tsx`・`frontend/src/components/analysis/AiRecognitionSection.tsx`。
- バックエンド変更なし・DBスキーマ変更なし・LLM呼び出し増なし（1解析=1チケット原則を維持）。
- 既存負債 `JobAnalysisPage.requestPdfReport`（未使用・PDFリクエスト配線漏れ）は今回スコープ外として温存。
