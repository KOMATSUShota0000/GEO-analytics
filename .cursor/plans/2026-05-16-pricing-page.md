# 仕様書: PricingPage 実装

## 目的

現状「準備中です。」の2行しかない `PricingPage.tsx` を、プラン比較表と問い合わせ／デモ申込み CTA を持つマーケティングページとして実装する。

`TierDiagnosisCard` の「プランを確認する」ボタンが `/pricing` へ誘導しているが、現状はそこで行き詰まりになっている。アップセルの入口（アップセルバナー）と出口（Pricing ページ）を接続し、SaaS グロースループを完成させることが目的。

Stripe 連携は本仕様のスコープ外。このページはあくまで「プランを比較して、問い合わせにつなげるマーケティングページ」として実装する。

## 対象ユーザー

- `TierDiagnosisCard` のアップセルバナーからページに来た Standard プランユーザー（SoM スコアが高くなり、より精密な分析を求めている）
- プラン体系を直接確認したい見込み顧客・既存顧客

## 機能要件

- Standard / Pro / Expert の3プラン比較表を表示すること
- 各プランの主要スペック（解析上限・競合エビデンス量・AI モデルアクセス・機能制限）を視覚的に比較できること
- Pro / Expert プランへの問い合わせ・デモ申込みへ誘導する CTA ボタンを設置すること（現時点ではボタン押下でメール起動 `mailto:` またはお問い合わせフォームへの外部リンクで可）
- ページ上部に「プランを確認する」から来たユーザー向けのコンテキスト説明文を表示すること（例：「GEO市場占有率が高まっています。さらなる精度向上のために」）
- `/pricing` へのルーティングが正しく機能していること（既存の `App.tsx` のルーティングを確認し、未定義なら追加する）
- 元のページ（`JobAnalysisPage`）に戻るための「解析に戻る」リンクを設置すること

## UI/UX 要件

### ページ構成

```
[ヘッダー]
  タイトル: 「GEO解析プランを選ぶ」
  サブコピー: 「LLM回答内での言及率・競合比較精度を最大化するプラン」

[コンテキストバナー]（Standard プランユーザー向け）
  「SoM スコアが向上しています。より精密な競合比較と100点満点評価で、次のGEO戦略を立案しましょう。」

[プラン比較カード]
  3カラム（モバイルはスタック）: Standard / Pro / Expert
  Pro カードを視覚的に強調（ボーダー・バッジ「おすすめ」）

[機能比較テーブル]
  行: 機能一覧
  列: Standard / Pro / Expert
  セル: ✓ / — / 数値

[CTA セクション]
  「プランのアップグレードについてお問い合わせください」

  「← ジョブ解析に戻る」
```

### プラン比較内容（`SubscriptionPlan.java` の定義に基づく）

| 機能 | Standard | Pro | Expert |
|------|----------|-----|--------|
| 1日の解析上限 | 10回 | 100回 | 200回 |
| 累計解析上限 | 10件 | 500件 | 2,000件 |
| リアルタイム一括上限 | 10件 | 50件 | 50件 |
| 競合エビデンス量 | 12,000文字 | 24,000文字 | 48,000文字 |
| 使用AIモデル | Gemini のみ | Gemini + ChatGPT | 全モデル（Claude含む） |
| Proプラン機能 | — | ✓ | ✓ |
| ホワイトラベル | ✓ | ✓ | ✓ |

### ビジュアルスタイル

- 既存のデザインシステム（Tailwind CSS + MUI）に準拠する
- ブランドカラー（`BrandingContext` の `brandColor`）をアクセントカラーに使用する
- `TierDiagnosisCard` と同様の `rounded-2xl border border-slate-200 bg-white shadow-sm` スタイルを踏襲する
- Pro カードのボーダーは `border-2 border-amber-400` で強調（アップセルバナーと同系色で統一感を出す）
- `LockedInsightCallout` の錠前アイコンや amber 系のカラーパレットを参考に、「Proでできること」を視覚的に表現する

### レスポンシブ

- デスクトップ: 3カラム横並び
- タブレット以下: 縦スタック（Standard → Pro → Expert の順）

## データ要件

- バックエンド連携は不要。プラン定義は `SubscriptionPlan.java` の値をフロントに静的定数として持たせる
- CTA ボタンの遷移先 URL は、実装時点では `mailto:` または仮の問い合わせフォームURL とする（将来的に Stripe へ切り替えやすいよう、リンク先を環境変数または設定値で切り替えられる構造にしておくことが望ましい）
- ブランドカラーは `useBranding` フックまたは `BrandingContext` から取得する

## プロダクトの核との整合性

- **WOW体験**: 直接の影響なし。ただし4人のペルソナ議論によって SoM スコアが明確に出たユーザーがアップグレードするきっかけになるため、WOW体験の価値を「次の行動」につなげる役割を持つ
- **実利**: ホワイトラベル対応の言及（「ブランドカラーでカスタマイズ可能」を比較表のホワイトラベル行に示す）で、代理店向けの訴求ができる
- **SaaSグロース**: このページがアップセルループの「出口」。TierDiagnosisCard（入口）→ PricingPage（比較・決断）→ 問い合わせ（CV）の流れを完成させる
- **高利益率**: LLM 呼び出しなし。静的ページのため追加コストゼロ
